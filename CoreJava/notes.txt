1.Exception Handling
  An Exception is an error that occurs during program execution and can stop the program.

   -Example:-
    int a=10; 
    int b=0;

    System.out.println(a/b);

   -Output:-
           ArithmeticException
           / by zero
           Program crashes.

    Why Exception Handling?
    Without Exception Handling:-

    System.out.println("Start");
    int a = 10/0;
    
    System.out.println("End");

    Output:-
    Start
    ArithmeticException
    End never prints because the program crashes.

   -try and catch

    try{
    int a = 10/0;
    }
    catch(Exception e){
    System.out.println("Error Occurred");
    }

    Output:
    Error Occurred
    Program does not crash.

    Meaning of every line:-

    try
        Put risky code here.

    int a = 10/0;
    This may create an exception.

    catch(Exception e)
    If an exception occurs, catch it.

    System.out.println("Error Occurred");
    Runs instead of crashing.

   -Example:-

     public class Main{

     public static void main(String[] args){

        System.out.println("Start");

        try{
            int a = 10/0;
        }
        catch(Exception e){
            System.out.println("Error Occurred");
        }

        System.out.println("End");
       }
     }

     Output:-
             Start
             Error Occurred
             End

     Step by Step Exection:-

     Step 1:-
     Java prints:
     Start

     Step 2:-
     Java enters:
     try{
        int a = 10/0;
     }

     Step 3:-
     Exception occurs:
     ArithmeticException

     Step 4:-
     Java immediately jumps to:
     catch(Exception e)

     Step 5:-
     Prints:-
     Error Occurred

     Step 6
     Program continues.
     Prints:
     End

3.Multiple Catch Blocks
  Different exceptions can be handled differently.

    -Example:-
     try{
       int[]arr={10,20,30};

      System.out.println(arr[5]);
     }
     catch(ArithmeticException e){
          System.out.println("Arithmetic Error");
     }
     catch(ArrayIndexOutOfBoundsException e){
          System.out.println("Array Error");
     }
     catch(Exception e){
          System.out.println("Some Error");
     }

    -Output:-Array Error

    -How Java Checks Catch Blocks
      Java checks from top to bottom.
    
      Exception Occurs
          ↓
      Check Catch 1
          ↓
      Check Catch 2
          ↓
      Check Catch 3
          ↓
      Matching Catch Executes

    -Important Rule
      Always keep:
      catch(Exception e)
      at the end.
      Because: Exception is the parent of most exceptions.

4.throw Keyword
  Sometimes we want to create an exception manually.

     -Example:
     
     throw new ArithmeticException("Invalid");
     
     Java immediately creates an exception.

     public class Main{

     public static void main(String[] args){

        throw new ArithmeticException("Invalid");
      }
     }

     -Output:
              ArithmeticException: Invalid
              Program crashes.

    -With try-catch

      public class Main{

      public static void main(String[] args){

        try{

            throw new ArithmeticException("Invalid");

        }
        catch(Exception e){

            System.out.println("Caught");
        }
     }
     }

    -Output:-
           Caught

        Step 1
        Java enters:
        try
        and starts executing code inside it.
        
        Step 2
        Java sees:
        throw new ArithmeticException();
        This means:
        Create an ArithmeticException RIGHT NOW.
        So yes, an ArithmeticException is created immediately.
        
        Step 3
        Now Java asks:
        Is there a matching catch block nearby?
        Java checks:
        catch(Exception e)
        and thinks:
        ArithmeticException IS-A Exception
        because:
        Exception
           ↑
        ArithmeticException
        ArithmeticException is a child of Exception.
        So this catch block can handle it.
        
        Step 4
        Java jumps directly to:
        catch(Exception e){
            System.out.println("Error");
            }
        and prints:
        Error
            
        Step 5
        After catch finishes, program continues:
        System.out.println("End");
        
        Output:
             Error
             End

       import java.io.IOException;

       public class Main{

       public static void test() throws IOException{
              throw new IOException();
        }

        public static void main(String[] args){

         }
        }

        Here:
          throws IOException

        means:
           test() may generate IOException

9.LinkedList
  A LinkedList is another collection that stores data in order like ArrayList.

   Why was LinkedList created?
   ArrayList is very good at:
   
   Reading data
   Accessing data
   But when inserting/deleting in the middle, Java has to shift elements.

   Example:

   Before:
          [10, 20, 30]

   Insert 15:
            [10, 15, 20, 30]

   Java shifts: 
               20
               30
    to the right.

    LinkedList was designed to make insertions and deletions easier.

   -Creating LinkedList
     Import:
     import java.util.LinkedList;

     Create:
     LinkedList<Integer> list=new LinkedList<>();

     Adding Elements:-
      list.add(10);
      list.add(20);
      list.add(30);

     Memory:-
       [10, 20, 30]

     Printing
       System.out.println(list);
       [10, 20, 30]

    -Methods are almost the same as ArrayList:
      add()
      get()
      remove()
      set()
      size()
      All work in LinkedList too.

    -Example:-
             LinkedList<Integer> list=new LinkedList<>();

             list.add(10);
             list.add(20);
             list.add(30);

             System.out.println(list.get(1));

    -Memory:-
             Index: 0 1 2
             Value:10 20 30
        
    -Output:-
             20

     ArrayList = Fast Access
     LinkedList = Fast Insert/Delete

   HashMap is a collection class that stores data in:
   key → Value pairs

   -Example:-
             101→Java
             102→HTML
             103→CSS

   -Creating HashMap:-

    import java.util.HashMap;
    HashMap<Integer,String> map = new HashMap<>();

   -Here:
        Integer → Key Type
        String  → Value Type
        map     → Variable Name

   -Adding Data:-
        map.put(101,"Java");
        map.put(102,"HTML");
        map.put(103,"CSS");

   -Memory:-
        101 → Java
        102 → HTML
        103 → CSS

   -Getting Data:-
        System.out.println(map.get(101));
        Java looks for:
               Key = 101
        and returns:
               Java
        Output:-
               Java

   -Finding Size:-
    System.out.println(map.size());

   -Output:- 
    3

   -Full code:-
    import java.util.HashMap;

    public class Main{
      public static void main(String[] args){
        HashMap<Integer,String> map = new HashMap<>();

        map.put(101,"Java");
        map.put(102,"HTML");
        map.put(103,"CSS");

        System.out.println(map.get(101));
        System.out.println(map.size());

      }
    }

   -Output:-
            Java
            3

   -Impotant Rule:-
    Hashmap can have duplicate keys
     -Example:
             map.put(101,"Java");
             map.put(101,"Python");

     -Now memory becomes:
             101 → Python
             Java value is replaced.
     -Output:
           System.out.println(map.get(101));
           Python

   -Difference:-
      -ArrayList
        Index → Value
        0 → Java
        1 → HTML
        2 → CSS

      -HashMap
        Key → Value
        101 → Java
        102 → HTML
        103 → CSS

   -Traversing a HashMap
    Just like we traversed an ArrayList:
     for(String name: list){
        System.out.println(name);
     }

      HashMap<Integer,String> map = new HashMap<>();

          statements

       Lots of code.

        }

       interface Demo{

       }

       public class Practice12{

        }
       }

    2.Functional Interface?
      A Functional Interface is an interface that contains exactly one abstract method.
        -Example:
         interface Demo{
           void show();
           }
           
          Here:
          show()
          is the only abstract method.
          So Demo is a Functional Interface.
          
          Why is it called Functional?
          Because Java 8 Lambda Expressions can work only with interfaces that have:
          Exactly 1 abstract method
          
          Valid Functional Interface
           interface Demo{
           void show();
           }
           
          Abstract methods:
          1
          Functional Interface
          
          Not a Functional Interface
          interface Demo{
            void show();
            void print();
            }
            Abstract methods:
            2
            Not a Functional Interface
            Lambda will not work here.

          -@FunctionalInterface Annotation
         
           Java provides:
           @FunctionalInterface
           interface Demo{
            void show();
            }
            
           This tells Java:
            "This interface must always have exactly one abstract method."
            If you accidentally add another abstract method:
            void print();
            Java gives a compile-time error.

          -Interview Questions
           Q1. What is a Functional Interface?
               A Functional Interface is an interface containing exactly one abstract method. It is mainly used with Lambda Expressions.
               
           Q2. Can a Functional Interface have
               It can have multiple methods, but only one abstract method. It may also contain default and static methods.

           Q3. Why are Functional Interfaces used?
               They enable Lambda Expressions and make code shorter and more readable.

    3.Stream API
      Stream API is used to process collections of data in a simple and efficient way.
      Operations:
                 Filtering
                 Sorting
                 Searching
                 Mapping
                 Counting

      -First Stream Example
       
        import java.util.*;

        public class Main{
         public static void main(String[] args){
           ArrayList<Integer> list = new ArrayList<>();

           list.add(10);
           list.add(15);
           list.add(20);
           list.add(25);

           list.stream()
              .filter(num -> num % 2 == 0)
              .forEach(System.out::println);
            
          }
         }

         -Output:-
          10
          20

        -Understanding Step-by-Step
          list.stream()
          Converts collection into a stream.
          [10,15,20,25]
                ↓
              Stream
              
          filter()
           Selects only matching elements.
           .filter(num -> num % 2 == 0)
           Checks: 
                 10 → Even 
                 15 → Odd 
                 20 → Even 
                 25 → Odd 
          Remaining:
                 10
                 20
                 
          forEach()
           Prints every element.
            .forEach(System.out::println);
            Output:
            10
            20

      -count():-
          Suppose we want to count how many even numbers exist.

          public class Main{
          public static void main(String[] args){
           ArrayList<Integer> list = new ArrayList<>();

           list.add(10);
           list.add(15);
           list.add(20);
           list.add(25);

           long count=list.stream()
                          .filter(num -> num % 2 == 0)
                          .count(); 

            System.out.println(count);
          }
         }

         Output:- 
         2 

      -map()
        filter() removes elements.
        map() transforms elements.

        Example:
        
          ArrayList<Integer> list = new ArrayList<>();
          
          list.add(1);
          list.add(2);
          list.add(3);
          
          list.stream()
              .map(num -> num * 10)

          Output:
                10
                20
                30
                
          Step-by-Step
           Original:
                    1
                    2
                    3
           Apply:
                 num * 10
                 
                 Result:
                        10
                        20
                        30
           Then:
               forEach()
               prints them.

          Interview Question
